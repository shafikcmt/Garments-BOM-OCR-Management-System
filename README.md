# Garments BOM / OCR Management System

## Project Overview

**Garments BOM / OCR Management System** is a centralized digital platform designed for garment manufacturing operations. It helps manage the full business workflow from **order creation to shipment**, replacing scattered manual Excel-based tracking with one organized and controlled system.

In many garment operations, teams depend on 40-50 separate Excel files to manage orders, BOM, purchase requirements, commercial documents, store updates, accounts activities, and shipment tracking. This creates duplicate work, slow communication, version confusion, and limited visibility across departments.

This system brings those processes into a single workspace where multiple departments can work together with clear responsibility, better control, and real-time operational visibility.

---

## Purpose of the System

The main purpose of this system is to simplify and control the garment order lifecycle by connecting key departments in one platform.

It supports business teams in managing:

- Order information
- Buyer and style details
- Bill of Materials (BOM)
- OCR-based document/data processing
- Purchase and PO-related workflow
- Store and inventory coordination
- Commercial documentation support
- Accounts-related control
- Shipment progress tracking
- Approval and role-based activity control
- Dashboard-based monitoring

---

## Business Problem It Solves

Garments production and export operations usually involve multiple departments working on the same order using different Excel files. This often causes:

- Repeated manual data entry
- High risk of mistakes
- Delayed communication between teams
- Missing or outdated files
- Difficulty tracking who changed what
- Poor visibility for management
- Slow reporting and decision-making
- Complex follow-up from order to shipment

This system reduces those issues by creating one controlled digital workflow where information can be entered, reviewed, updated, and tracked from a single source.

## Key Departments Supported

The system is designed for cross-department collaboration, including:

### Merchandising Team
Manages order details, style information, BOM preparation, buyer requirements, and order progress follow-up.

### Commercial Team
Supports export/import-related documentation flow, commercial data handling, and shipment-related coordination.

### Store Team
Tracks material receiving, availability, issue status, and inventory-related updates.

### Accounts Team
Supports cost, payment, PO, and finance-related monitoring based on approved workflow data.

### Management / Admin Team
Controls user roles, permissions, workflow access, approval visibility, dashboards, and reporting views.

---

## Core Features

### Centralized Order Management
Manage garment orders in one place with structured information, status tracking, and department-wise visibility.

### BOM Management
Prepare, review, and control Bill of Materials data for styles, items, colors, sizes, quantities, and related requirements.

### OCR-Based Data Assistance
Use OCR-supported document handling to reduce manual typing and improve document-to-system data entry efficiency.

### PO and Purchase Control
Manage purchase-related information, PO generation workflow, and approval-based control for responsible teams.

### Role-Based Access Control
Each department receives access based on responsibility. Admin can control who can view, create, update, approve, or manage specific parts of the system.

### Workflow Monitoring
Track process status from order stage to shipment stage with better visibility and accountability.

### Dashboard and Reports
Provide management-friendly dashboards for order progress, pending tasks, PO status, shipment updates, and department-wise activities.

### History and Change Visibility
Maintain better control over updates, changes, and process movement so teams can understand the latest status clearly.
