# PDF / Excel / Export Rules

PDF, Excel, and export files are official business documents. Treat them carefully.

## General Rules

- Match reference image/file when provided.
- Keep paper size and orientation correct.
- Preserve official header, logo, table structure, signature area, and date format.
- Use compact spacing where needed.
- Avoid broken page breaks.
- Avoid content cutting.
- Preview output and download output should match.

## PDF Layout Checklist

Before finalizing PDF changes, check:

- Page size: A4 or requested size
- Orientation: portrait/landscape
- Header alignment
- Logo size
- Font size
- Font weight
- Table header names
- Column widths
- Border thickness
- Signature/date area
- Top/bottom spacing
- Page break behavior
- Long text wrapping
- Multi-page output

## Excel Export Checklist

Before finalizing Excel changes, check:

- Correct column order
- Correct heading names
- Correct merged cells if used
- Correct date/number formatting
- No missing required column
- Totals/subtotals correct
- Exported file opens without warning
- Filters and formatting do not break data

## Reference File Rules

When the user provides screenshot/reference:

1. Compare current vs reference.
2. Identify only the mismatch.
3. Change only necessary CSS/HTML/PDF template code.
4. Do not redesign unless the user asks.
5. Explain exactly what was adjusted.

## Common Export Risk

- Data shown in preview but missing in download
- Column name changed in view but export still old
- Large text breaks table
- Multiple selected records create wrong PO number
- Date auto-generation not applied consistently
- Signature section too close to table
